# 指示 (Instruction)

あなたは小説家です。指定された設定とプロット、要件に基づき、**このシーンの本文を執筆してください。**
まずはこれまでに執筆された小説本文である Story So Far (Full Text) を提示し、その後に詳細のプロンプト、要件を提示します。

# Input Data
## Story So Far (Full Text)
ここには、これまでに執筆された小説本文がそのまま入ります。
Scene Setup で指示されている場合を除き、ここでの情報を再びリフレインする必要はありません。
必要に応じて参照し、登場人物の口調・描写の一貫性・伏線などを保ってください。
過去にすでに描かれた出来事を再び扱う場合は、「新しい情報・新しい感情・新しい視点」が必ず追加されるようにしてください。

<story-so-far>
{{story_so_far_full_text}}
</story-so-far>

以上までが執筆済みの Story So Far (Full Text) でした。

# 要件
**最重要: 文体ガイドラインへの準拠**
下記「Style Guide（文体ガイドライン）」に記載された作家ペルソナ・文体・禁止事項を**厳格に守って**執筆してください。これはこの物語全体の文体を統一するための指針です。

**要件:**
1.  **文体**: 「Style Guide」で定義された作家ペルソナとして振る舞い、指定された文体・語彙・修辞・禁止事項に従うこと。
2.  **分量**: 600文字程度〜2,000文字まで。
3.  **描写**: 五感（視覚、聴覚、触覚、嗅覚など）を使った具体的な描写を入れること。
4.  **Show, Don't Tell**: 「彼は悲しかった」と説明するだけではなく、できれば行動や表情・仕草・周囲の反応を通じて悲しみを表現すること。`Scene Setup` に心情やテーマの説明が含まれていても、本文ではそのまま言い直さず、必ず「出来事」と「描写」に変換すること。
5.  **会話と地の文のバランス**: テンポの良い会話と、没入感のある地の文を組み合わせる。
6.  **出力**: 下記のフォーマットに厳密に従ってください。
7.  **コンテキストの使い方**:
    * 「Story So Far (Full Text)」にすでに描かれている出来事や説明を、そのまま長く繰り返さないこと。必要な場合でも、1〜2文の短い振り返り＋新しい視点やリアクションに留めてください。
    * 今回のシーンで何を書くべきかは、必ず `Scene Setup`（`scene_intent_and_events`）の指示を最優先してください。ただし、`Scene Setup` に書かれているのは「起こるべき出来事とドラマの要約」であり、「そのまま本文として語るべき文章」ではありません。`Scene Setup` の内容を、登場人物の行動・会話・具体的な描写として再構成してください。

# 出力フォーマット指示
出力は必ず次の2つのセクションのみで構成してください。これ以外の見出しや余計なテキストは出力しないでください。

```markdown
# 本文
（ここに小説本文を記述する。章番号や見出しは不要。必要であれば、この中で `##` 以下の見出しを使っても良いが、`# 本文` より上位の見出しは追加しないこと。）

# 次のシーンで描くこと
（ここに、次のシーンで描きたいこと・続きで掘り下げるべき要素・残したい余韻などを、LLM が迷わないように簡潔にメモとして記述する。読者向けの本文ではなく、あくまで「次のシーン生成のための意図 (scene intent)」として書くこと。）
```
